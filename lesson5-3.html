Building a Network


Building a network involves establishing connections between devices, systems, or individuals to enable communication and data exchange. This process can be physical, such as wiring computers through routers and switches, or virtual, through software configurations and cloud services. A well-structured network ensures reliable access, security, and efficient sharing of resources across a local area network (LAN) or a wide area network (WAN). It requires careful planning, proper hardware, and security measures to support the desired functionality and scalability.

Connecting to the Internet: Wireless & Ethernet

- There are several methods to connect to the Internet, with wireless and wired (Ethernet) being the most common.

Wireless Network Connection:
- Secured Networks: Most Wi-Fi networks are secured with a password. To connect to a secured network, you'll need to obtain the password from the Network Administrator
or designated person (e.g., at a coffee shop or hotel).
- Unsecured Networks: Some networks are open, meaning anyone can connect without a password, like in public places (e.g., coffee shops).
- Signal Strength: A secure network is often marked with a shield icon on the signal strength indicator. Stronger signals are represented by more radio wave bars, while
fewer bars indicate weaker signals.

Wired Network Connection:
- Wired connections (Ethernet) are indicated by a different icon compared to wireless networks. To connect, simply select the desired network and enter a password if needed.
- In both cases, once you identify the network, you can select Connect and proceed with the necessary steps.

        Notes:
        * Before connecting, make sure your wireless adapter is turned on – a radio icon usually represents this.
        * Some computers will require a keyboard command; typically the function key and an embedded key to enable it, or a simple dedicated wireless button.

Checking The Connectivity of the Two Computers

- To diagnose network issues and verify connectivity between computers, various tools and protocols are used.


ICMP (Internet Control Message Protocol)

- A network layer protocol used by devices to diagnose network communication issues.
- Function: ICMP helps determine if data is reaching its destination on time and communicates network issues back to the source.
- Key Messages: It sends control messages such as "destination network unreachable," "source route failed," and "source quench."
- Packet Structure: It uses an 8-byte header with a variable-size data section.


Network Diagnostic Commands
    
1. IPCONFIG

    - A command-line utility in Windows that displays TCP/IP network configuration.
    - Function: It shows IPv4 and IPv6 addresses, subnet mask, and default gateway. It also refreshes DHCP and DNS settings.
    - Usage: Running ipconfig without parameters displays the network configuration for all adapters.

    
2. PING

    - A network utility to test if a particular host is reachable across an IP network and measure the round-trip time for packets.
    - Results:
        - Reply From: Indicates a successful connection.
        - Request Timed Out: Indicates failure to connect.
    
    
3. (CMD) Command Prompt

    - The default command-line interface for Windows operating systems.
    - Function: Allows users to run commands and perform system management tasks.
    - Usage: Opened by typing cmd in the Windows search bar.

    
4. DOS Prompt

    - A command-line indicator in MS-DOS (Microsoft Disk Operating System).
    - Default Prompt: The current drive letter followed by a greater-than sign (C:\> for the C drive).

Error Results in Pinging

Request timed out

- It means that the ICMP packet reached from one host to the other host but the reply could not reach to the requesting host.
- There may be more packet loss or some physical issues.

Destination host unreachable

- It means there is no proper route defined between two hosts.
- There are many possible reasons for getting a “destination host unreachable” error, including things as simple as erroneously connected cables or an overly aggressive firewall.

General failure

- The Command Prompt does not provide any additional information as to exactly what failed or why the computer failed to run the ping command.
- That being the case, there are a ton of different possible causes for this problem.


Building a Switched-Based Network

- When expanding an existing computer network or building a new one, one of the many devices that can be used in the process is an Ethernet Switch.
- A Switch is a simple device that connects multiple computers together and to the rest of the network, allowing communication to occur between all connected devices.

The following questions should be addressed first before setting up the network:
- How many computers are you going to connect to?
- Do you have a cable run already or are you going to terminate one for yourself?
- What is the distance between computers in the network?
- What type of operating system are you going to use in your network?
- These are some of the common questions that would be answered first so that no problems may arise during the installation.

To Set-up a Network Using a Switch

1. Be ready with the cable that you are going to use. Since we will be using a switch, we are connecting two different devices, so a STRAIGHT THROUGH cable is advisable.
            

2. Set up the switch in the middle of each computer; do not forget to power it up.  
            

3. Connect the cable of each computer’s NIC to the Switch ports.
            

4. Power up your computer and wait for several seconds for the booting process.
            

5. Configure the TCP/IP Settings of each computer and give the individual IP Address and Subnet Mask.

How To Set Up A Wireless LAN?

- A WLAN is a network that typically covers a small area, such as a home, office, or public places like libraries, coffee shops, or parks.

Advantages of WLAN:
- Lower Setup Cost: WLANs are generally more affordable to set up than wired networks.
- Ease of Setup: Modern wireless routers are easy to install due to their plug-and-play functionality.

Setup Process:
- Buy a Wireless Router: Popular brands include Linksys, Belkin, and D-Link.
- Compatibility: The router should support various wireless cards and allow easy connection for users with different devices, including older
wireless protocols. Most modern routers are backward compatible.

Basic IP Addressing

- IP addressing is important because it facilitates the primary function of the Internet Protocol (IP) — the delivery of datagram across an internetwork.
- IP Addressing scheme makes it possible for computers to “find each other” online and exchange information.
- Understanding this in more detail requires us to examine a few different but essential issues related to how IP and its addresses operate.

IP Address Functions: Identification and Routing

There are actually two different functions of the IP address namely:


Network Interface Identification

- Internet Protocol requires an address to route traffic to the correct destination.
- Just like a postal address at your house helps the post office deliver mail to a specific destination, an IP address is used by the routers to deliver your email and your web data to you.


Routing

- When the source and destination of an IP datagram are not on the same network, the datagram must be delivered “indirectly” using intermediate systems, a process called routing.
- The IP address is an essential part of the system used to route datagram.

Private IP Address Versus Public IP Address

What is Public IP Address?
- An Internet Protocol (IP) address that is designated for use in a public domain**, such as the Internet.

What is Private IP Address?
- These addresses can be used on a private network, but they are not routable through the public Internet.
- This not only creates a measure of much-needed security, but it also conveniently saves valuable IP address space.

        Note:
        - These are known as **private IP addresses** and are found in the following ranges:
                > From 10.0.0.0 to 10.255.255.255
                > From 172.16.0.0 to 172.31.255.255
                > From 192.168.0.0 to 192.168.255.255

IP Address

- An IP (Internet Protocol) address is a unique identifier for a node or host connection on an IP network.
- An IP address is a 32-bit binary number
        - usually represented as 4 decimal values
        - each decimal value is representing 8 bits, in the range 0 to 255 (known as octets)
        - separated by decimal point (.)
        - known as "dotted decimal" notation
        - Example: 192.168.1.1

IP Addresses are useful if it is viewed in its binary form as follows:



Every IP address consists of two parts:
- one identifying the network
- one identifying the host

The Class of the address and the subnet mask determine which part belongs to the network address and which part belongs to the host address.

Address Classes

- There are five (5) different address classes.
- You can determine which class any IP address is in by examining the first 4 bits of the IP address or the first octet.



Network ID determines the Network portion (N) and the Host portion (H) of the address.
        > Network Part
        - The network part  specifies the unique number that is assigned to your network
        - The network part also identifies the class of network that is assigned.
        
        > Host Part
        - This is the part of the IPv4 address that you assign to each host
        - The host part uniquely identifies the machine on your network
        
            Important Notes:
            - For each host on your network, the network part of the address is the same, but the host part must be different
            - You can test this:
                  - You should always be able to ping 127.0.0.1, which points to yourself
                    - Class D addresses are reserved for multicasting
                    - Class E addresses are reserved for future use (they should not be used for host addresses)
            - Addresses beginning with 01111111, or 127 decimal, are reserved for loopback and for internal testing on a local machine

Subnet Mask

- A subnet mask is a number that defines a range of IP addresses available within a network.
- A single subnet mask limits the number of valid IPs for a specific network.
- Multiple subnet masks can organize a single network into smaller networks (called subnetworks or subnets).
- Systems within the same subnet can communicate directly with each other, while systems on different subnets must communicate through a router.

Gateway Address

- The gateway address (or default gateway) is a router interface connected to the local network that sends packets out of the local network.
- The gateway has a physical and logical address.


            Note: Network professionals use the First or Last IP Address in a particular network or subnetwork as Gateway addresses.
