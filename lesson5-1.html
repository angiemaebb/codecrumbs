Computer Network


Computer networking refers to the interconnection of multiple computing devices, such as computers, servers, smartphones, and other hardware, to share resources and information efficiently. Networks enable communication, data exchange, and resource sharing, making them fundamental to modern technology.

Computer Network

A computer network consists of devices like computers, servers, and peripherals connected to share data and resources. Networks can range from simple, like 
two computers connected by a cable, to complex systems like the Internet. Devices such as hubs or wireless communication facilitate connections, enabling data exchange.

Simple Network

A network is more than just a collection of computers; it creates new capabilities by offering connectivity and sharing. Networking allows for shared resources and
facilitates communication, which enhances productivity beyond individual machine capabilities.



Advantages of Networking

1. Connectivity and Communication: Enables communication via emails, chats, and other technologies, fostering collaboration within LANs.
2. Data Sharing: Facilitates easy and quick data sharing, replacing methods like "sneakernet."
3. Hardware Sharing: Allows multiple users to share devices like printers.
4. Internet Access: Provides access to the global network and the web.
5. Data Security and Management: Centralized data makes management easier and more secure.
6. Entertainment: Supports multi-player games and media streaming over LANs or the Internet.


Disadvantages of Networking

1. Virus and Malware: Networks make it easier for malicious software to spread.
2. Vulnerabilities: Networks create potential new entry points for unauthorized access.
3. Complexity: Setting up and managing networks requires expertise.
4. Cost: Initial setup and ongoing maintenance can be costly.
5. Security Risks: Poorly secured networks expose data to hackers or sabotage.


Different Categories of Network

(LAN) Local Area Network

Connects computers within a small geographic area. It supports high-speed data transfer and can be organized in peer-to-peer or client/server modes.



(MAN) Metropolitan Area Network

Links multiple LANs over a few dozen kilometers, using high-speed connections.



(WAN) Wide Area Network

Connects LANs across large distances, with varying speeds depending on cost and infrastructure.




Client-server Architecture

In client-server networks, the server provides resources or services, while client computers request and receive information from the server.
Servers handle central processing tasks, and clients rely on them for these services.


Different Types of Networking

1. The Intranet

A private network using Internet technologies but isolated from the global Internet. It is secure and often used for internal communication and resources
within organizations.

2. The Extranet

An extension of the intranet, providing access to authorized external parties like partners or suppliers, typically through VPNs, for secure
business-to-business operations.

3. The Internet

A vast global network connecting millions of private, public, and organizational networks, offering a wide range of services, from email to social
networking and online shopping.


Network Topology

Network topology defines the physical and logical arrangement of network elements, affecting the system's efficiency and reliability. Common topologies include:


1. Star Topology

All devices connect to a central hub or switch. It’s easy to set up, but if the hub fails, the whole network is affected.

> Advantages:
- If N devices are connected to each other in a star topology, then the number of cables required to connect them is N.
    - So, it is easy to set up
- Each device requires only 1 port (i.e., to connect to the hub).

> Disadvantages:
- If the concentrator (hub) on which the whole topology relies fails, the whole system will crash down.
- The cost of installation is high.
- Performance is based on the single concentrator (i.e., hub).

2. Bus Topology

Devices share a single backbone cable. It's cost-effective but vulnerable to network crashes if the cable fails.

> Advantages:
- If N devices are connected to each other in a bus topology, then the number of cables required to connect them is 1 which is known as backbone cable, and N drop lines are required.
- The cost of the cable is less as compared to other topology, but it is used to build small networks.

> Disadvantages:
- If the common cable fails, then the whole system will crash down.
- If the network traffic is heavy, it increases collisions in the network.
    - To avoid this, various protocols are used in the Media Access Control (MAC) layer known as Pure Aloha, Slotted Aloha, Carrier Sense Multiple Access/Collision Detection (CSMA/CD), etc.

3. Ring Topology

Devices form a closed-loop. It uses a token to transfer data, minimizing collisions, but troubleshooting can be difficult.

There are two types of token release techniques:
A. Early token release releases the token just after the transmitting the data
B. Delay token release releases the token after the acknowledgment is received from the receiver.

> Advantages:
- The possibility of collision is minimum in this type of topology.
- Cheap to install and expand.

> Disadvantages:
- Troubleshooting is difficult in this topology.
- The addition of stations in between or removal of stations can disturb the whole topology.

4. Mesh Topology

Each device connects to every other device with dedicated channels, offering high security and reliability but is costly and complex to install.

> Advantages:
- It is robust.
- The fault is diagnosed easily.
- Data are reliable because data are transferred among the devices through dedicated channels or links.
- Provides security and privacy.

> Disadvantages:
- Installation and configuration is difficult.
- Cost of cables are high as bulk wiring is required, hence, suitable for fewer number of devices.
- Cost of maintenance is high.

5. Hybrid Topology

Combines multiple topologies for flexibility, scalability, and fault tolerance, though it’s more complex and expensive to design.

> Advantages:
- Reliable: Fault detection and troubleshooting are easier, isolating issues without affecting the whole network.
- Scalable: Easily expandable without disrupting existing architecture.
- Flexible: Can be tailored to meet organizational needs and optimize resources.
- Effective: Combines strengths of multiple topologies, maximizing benefits and minimizing weaknesses.

> Disadvantages:
- Complex Design: Designing and configuring the network is challenging and requires expertise.
- Costly Hub: Specialized, expensive hubs are needed to connect different topologies.
- Expensive Infrastructure: Requires substantial investment in cables, cooling, and sophisticated devices.
